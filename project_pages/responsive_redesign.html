Responsive Redesign

October 18, 2022

SUMMARY

Responsive Redesign is a project practicing the workflow of redesigning a simple website, from identifying flaws in an existing interface to creating low and high fidelity prototypes to building the site itself. The first part of this project, Identifying Usability Problems, involved analyzing the usability and accessibility of the. The second part of this project, Visual Redesign, involved creating low-fidelity wireframes, a visual design style guide, and high-fidelity prototypes in order to address any issues identified in the first section. The final part of this project, Responsive Redesign, involved using the final high-fidelity prototypes to create the page using HTML and CSS.

PART 1: IDENTIFYING USABILITY PROBLEMS

Picking a Web Page

I chose the site for Small Point Cafe. A while ago, my friends and I were looking for coffee shops in the area to explore, and we found Small Point Cafe. We thought it looked like a cute cafe, so we decided to look up the menu before we went. We had a hard time navigating the site, and while we eventually found the menu, we were so annoyed at the site itself that we just chose another coffee shop. When I was thinking about websites for this project, I remembered disliking this particular one and thought it would be a good idea to redesign it.

Finding Problems

Pages Linked from the Home Page

Learnability
  • 3 different styles of pages containing different information and links (menu, contact, hours, etc.) making it hard to locate anything quickly
  • each of the different styles of pages have different navbars containing different links to pages
  • some items on the menu don't have descriptions about what is in the dish, so new users may not know what to get
  • home page has a link for "order for delivery" that takes users to their menu and allows them to place an online order which is not intuitive if users just want to look at menu items
Memorability
  • different styles of pages that contain different information makes it difficult to remember which links to click to get you to the desired destination
  • menu page does not contain a link back to the home page, so users would need to remember what pages to go through to return to the home page
  • navbar of the menu page has an option for "order for delivery" and a dropdown for "more" that includes "student snackpass", "order delivery", or "purchase a giftcard" rather than having all of these options easily visible
Efficiency
  • 3 different style pages makes it incredibly frustrating to navigate through the site quickly
  • even having used the website before, I still had to try a few different times to get to the menu page
  • order delivery page just contains links to the DASAP and GrubHub forcing users to navigate through a few pages to first look at the the menu and then order
Other
  • all images in the slideshow of the home page are cut off, and the slideshow does not allow for user scrolling
  • spelling mistakes (e.g. on the navbar of one of the page styles, "delivery" is spelled "delivey")
  • home page has inconsistent capitalization of the different navigation links
  • menu page contains duplicates of some food items
  • home page conains circles to separate different links, but resizing the screen makes them look like bullet points

Accessibility

For the home page, the WAVE tool doesn't point out any errors or contrast errors on the page. This makes sense given that page mainly consists of images, black text on a white background, and functioning links. There are, however, quite a few warnings. These warnings primarly concern the alternative text of the images. Specifically, many of the images in the slideshow have the same alternative text, and all of the images on the page (the ones in the slideshow and the logo) have alternative text that insufficiently describes the image/its purpose.

PART 3: RESPONSIVE REDESIGN

The newly designed site can be found here. The buttons on the left side navigation bar do work, so you can click on those and it will take you to the respective pages. The only page that is not designed like the prototypes is the MENU page simply because I did not have the capacity to create the page the way I wanted, so there is a picture of a capybara instead :). I know we were only required to implement the home page, but for this design, the home page seemed a bit simple, so I decided to implement a couple other pages. Overall, if I were to do this project again, I would definitely implement the MENU page since I think there were quite a few issues with that page.
